# Sélecteurs descendants

CSSXpath?
h1//h1?
div p//div//p?
ul > li//ul/li?
ul > li > a//ul/li/a
div > *//div/*
:root/?
:root > body/body

# Sélecteurs d’attribut

CSSXpath?
#id//*[@id="id"]?
.class//*[@class="class"] ...kinda
input[type="submit"]//input[@type="submit"]
a#abc[for="xyz"]//a[@id="abc"][@for="xyz"]?
a[rel]//a[@rel]
a[href^='/']//a[starts-with(@href, '/')]?
a[href$='pdf']//a[ends-with(@href, '.pdf')]
a[href*='://']//a[contains(@href, '://')]
a[rel~='help']//a[contains(@rel, 'help')] ...kinda

# Sélecteurs d’ordre

CSSXpath?
ul > li:first-of-type//ul/li[1]?
ul > li:nth-of-type(2)//ul/li[2]
ul > li:last-of-type//ul/li[last()]
li#id:first-of-type//li[1][@id="id"]?
a:first-child//*[1][name()="a"]
a:last-child//*[last()][name()="a"]

# Frères (siblings)

CSSXpath?
h1 ~ ul//h1/following-sibling::ul?
h1 + ul//h1/following-sibling::ul[1]
h1 ~ #id//h1/following-sibling::[@id="id"]

# jQuery

CSSXpath?
$('ul > li').parent()//ul/li/..?
$('li').closest('section')//li/ancestor-or-self::section
$('a').attr('href')//a/@href?
$('span').text()//span/text()

# Autres

CSSXpath?
h1:not([id])//h1[not(@id)]?
Correspondance texte//button[text()="Submit"]?
Correspondance (sous‑chaîne)//button[contains(text(),"Go")]
Comparaison numérique//product[@price > 2.50]
A des enfants//ul[*]
A un enfant spécifique//ul[li]
Logique OU//a[@name or @href]?
Union (fusionne les résultats)//a | //div?

# Class check

//div[contains(concat(' ',normalize-space(@class),' '),' foobar ')]

Xpath n’a pas d’opérateur pour « vérifier l’appartenance à une liste séparée par espaces ». Cette forme est le contournement courant.

Séparez les étapes avec /. Utilisez // si vous ne voulez pas seulement les enfants directs.

# Étapes

//div
//div[@name='box']
//[@id='link']

Une étape peut avoir un nom d’élément (div) et des prédicats ([...]). Les deux sont optionnels. Elle peut aussi être :

//a/text()     #=> "Go home"
//a/@href      #=> "index.html"
//a/*          #=> All a's child elements

# Prédicats

# Prédicats

//div[true()]
//div[@class="head"]
//div[@class="head"][@id="top"]

Restreint le nodeset uniquement si la condition est vraie. Les prédicats peuvent être chaînés.

# Opérateurs

# Comparison
//a[@id = "xyz"]
//a[@id != "xyz"]
//a[@price > 25]

# Logic (and/or)
//div[@id="head" and position()=2]
//div[(x and y) or not(z)]

Utilisez les opérateurs de comparaison et de logique pour construire des conditions.

# Utiliser des nœuds

# Use them inside functions
//ul[count(li) > 2]
//ul[count(li[@class='hide']) > 0]

# This returns `<ul>` that has a `<li>` child
//ul[li]

Vous pouvez utiliser des nœuds dans les prédicats.

# Indexation

//a[1]                  # first <a>
//a[last()]             # last <a>
//ol/li[2]              # second <li>
//ol/li[position()=2]   # same as above
//ol/li[position()>1]   # :not(:first-of-type)

Utilisez [] avec un nombre, ou last() ou position().

# Ordre d’enchaînement

a[1][@href='/']
a[@href='/'][1]

L’ordre compte : ces deux expressions sont différentes.

# Prédicats imbriqués

//section[.//h1[@id='hi']]

Retourne <section> s’il a un <h1> descendant avec id='hi'.

# Axes

# Utiliser les axes

//ul/li                       # ul > li
//ul/child::li                # ul > li (same)
//ul/following-sibling::li    # ul ~ li
//ul/descendant-or-self::li   # ul li
//ul/ancestor-or-self::li     # $('ul').closest('li')

Les étapes sont séparées par / pour sélectionner des enfants, mais vous pouvez spécifier un autre axe avec ::.

//ul/child::li
AxeÉtapeAxeÉtape

# Axe child

# both the same
//ul/li/a
//child::ul/child::li/child::a

child:: est l’axe par défaut. C’est pourquoi //a/b/c fonctionne.

# both the same
# this works because `child::li` is truthy, so the predicate succeeds
//ul[li]
//ul[child::li]

# both the same
//ul[count(li) > 2]
//ul[count(child::li) > 2]

# Axe descendant-or-self

# both the same
//div//h4
//div/descendant-or-self::h4

// est un raccourci de l’axe descendant-or-self::.

# both the same
//ul//[last()]
//ul/descendant-or-self::[last()]

# Trouver un parent

//section[h1[@id='section-name']]

Trouve un <section> qui contient directement h1#section-name.

//section[//h1[@id='section-name']]

Trouve un <section> qui contient h1#section-name. (Même résultat, mais avec descendant-or-self au lieu de child)

# Closest

./ancestor-or-self::[@class="box"]

Équivaut à $().closest('.box') de jQuery.

# Attributs

//item[@price > 2*@discount]

Trouve <item> et vérifie ses attributs.
